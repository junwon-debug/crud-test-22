# CLAUDE.md

이 파일은 이 저장소에서 작업하는 Claude Code(및 기타 AI 코딩 어시스턴트)를 위한 가이드입니다.

## 프로젝트 개요

JSON 데이터를 파싱하고, 이를 다시 JSON 파일로 저장하는 기능을 제공하는 라이브러리를 만드는 **POC(Proof of Concept)** 프로젝트입니다.
아직 요구사항과 구조가 확정되지 않은 초기 단계이며, 핵심 동작을 빠르게 검증하는 것이 목표입니다.

> **현재 단계: TDD 구현 단계입니다.** 설계(요구사항/데이터 모델/함수 시그니처)는 정리되었고, `tests/`에 실패 테스트를 먼저 작성한 뒤 `src/poc_crud_lib`에 최소 구현을 추가하는 방식으로 CRUD 함수를 하나씩 구현합니다. (Create/Read/Update 완료, Delete 진행 예정)

## 목표 (POC 범위) — CRUD 요구사항

JSON 파일을 저장소로 사용하는 CRUD 기능을 제공합니다.

- **Create**: 새로운 데이터를 입력 받아 JSON 파일에 저장한다.
- **Read**:
  - 전체 목록을 조회할 수 있다.
  - 특정 ID/키 값으로 개별 데이터를 검색할 수 있다.
- **Update**: 기존 데이터 중 하나를 선택하여 특정 필드를 수정할 수 있다.
- **Delete**: 특정 데이터를 안전하게 삭제할 수 있다(삭제 전 대상 존재 여부 확인, 원본 파일 손상 방지).

### 데이터 모델 / 저장 포맷

- **ID 체계**: 단순 incremental(자동 증가) 정수. Create 시 사용자가 id를 직접 지정하지 않으며, 기존 최대 id + 1을 자동으로 부여한다. 삭제된 id는 재사용하지 않는다.
- **필드 구성**: 고정 스키마 없이 **자유 형식(dict)**. `id`를 제외한 나머지 필드는 사용자가 Create 시 넘긴 대로 그대로 저장한다(필드명/타입 제약 없음).
- **저장 포맷**: JSON 파일 최상위는 **리스트(배열) 구조**로 각 항목(레코드)을 담는다.

  ```json
  [
    { "id": 1, "name": "foo", "value": 123 },
    { "id": 2, "tag": "bar" }
  ]
  ```

### CRUD 함수 시그니처

클래스 없이 **함수 단위**로 제공한다(POC 범위에서 상태를 들고 있는 객체/클래스는 불필요). 모든 함수는 대상 JSON 파일 경로를 인자로 받아 그때그때 읽고 쓴다.

```python
def create(file_path: str, data: dict) -> dict:
    """data를 저장하고, id가 부여된 레코드를 반환한다."""

def read_all(file_path: str) -> list[dict]:
    """전체 레코드 목록을 반환한다."""

def read_by_id(file_path: str, id: int) -> dict | None:
    """id에 해당하는 레코드를 반환한다. 없으면 None."""

def update(file_path: str, id: int, fields: dict) -> dict:
    """id에 해당하는 레코드의 필드를 fields로 갱신하고 결과를 반환한다.
    id가 없으면 ValueError를 발생시킨다."""

def delete(file_path: str, id: int) -> None:
    """id에 해당하는 레코드를 삭제한다.
    id가 없으면 ValueError를 발생시킨다."""
```

- 존재하지 않는 id에 대한 `update`/`delete`는 `ValueError`를 raise한다(POC 단계에서 별도 커스텀 예외 클래스는 만들지 않는다).
- `create`는 `id` 필드가 이미 포함된 `data`를 받더라도 무시하고 자동 부여한 id로 덮어쓴다.
- `update`의 `fields`는 부분 갱신(patch)이다 — 넘긴 키만 갱신하고 나머지 필드는 유지한다.

> 함수/API 시그니처는 위와 같이 잠정 확정되었습니다. 구현 중 세부 사항(예외 타입 등)이 바뀌면 이 섹션도 갱신합니다.

## 범위 밖 (Out of Scope, 현재 기준)

- 대용량 스트리밍 파싱, 성능 최적화
- 스키마 검증(JSON Schema 등) — 필요 시 추후 확장
- 동시성/락 처리 등 멀티 프로세스 환경 대응

## 기술 스택

- 언어/런타임: Python
- 패키지 매니저: uv
- 테스트 프레임워크: pytest
- 버전 관리: git

## 개발 방식

- **TDD(Test-Driven Development)** 로 진행합니다. 항상 실패하는 테스트를 먼저 작성한 뒤, 이를 통과시키는 최소한의 구현을 추가합니다.
- 테스트 실행은 `uv run pytest`를 사용합니다.
- 의존성 추가/설치는 `uv add`, `uv sync` 등 uv 명령을 사용합니다(pip 직접 사용 지양).

## 개발 규칙

- 이 프로젝트는 POC이므로 **오버엔지니어링을 절대 지양**합니다.
  - 지금 필요하지 않은 기능, 설정, 확장 포인트는 만들지 않습니다.
  - 추상 클래스/인터페이스, 플러그인 구조, 범용 설계 등은 실제로 두 번째 사용처가 생기기 전까지 도입하지 않습니다.
  - "나중에 필요할 것 같다"는 이유로 코드를 추가하지 않습니다.
- 과도한 추상화나 확장성 설계보다는 **동작하는 최소 구현**을 우선합니다.
- 파일 저장 시 원본 데이터 손상을 방지하기 위해 쓰기 실패/중단 상황을 고려합니다(예: 임시 파일에 쓴 후 교체).
- 외부 의존성은 꼭 필요한 경우에만 추가합니다.

## 디렉토리 구조

POC 성격에 맞게 최대한 단순한 `src` 레이아웃을 사용한다.

```
poc-crud-lib/
├── CLAUDE.md
├── pyproject.toml
├── .gitignore
├── src/
│   └── poc_crud_lib/
│       └── __init__.py     # create/read_all/read_by_id/update/delete 구현 위치
└── tests/                  # pytest 테스트 (TDD로 여기부터 작성 시작)
```

- 별도의 `cli/`, `config/`, `utils/` 등은 실제로 필요해지기 전까지 만들지 않는다.
- 모듈이 커지면 `poc_crud_lib/` 내부에 파일을 나누되, 지금은 `__init__.py` 하나로 시작한다.
